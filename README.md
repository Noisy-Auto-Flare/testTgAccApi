# Проверка Telegram API аккаунта

Скрипт проверяет работоспособность Telegram API: подключается к Telegram, выполняет `get_me()` и выводит данные аккаунта или сообщение об ошибке.

## Требования

- Python 3.8+
- [Telethon](https://docs.telethon.dev/)

## Установка

```bash
pip install -r requirements.txt
```

## Получение api_id и api_hash

1. Зайдите на [my.telegram.org](https://my.telegram.org).
2. Войдите по номеру телефона.
3. Создайте приложение в разделе "API development tools".
4. Скопируйте **api_id** (число) и **api_hash** (строка).

## Запуск

### Через .env (рекомендуется)

1. Скопируйте `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
2. Откройте `.env` и подставьте свои `TELEGRAM_API_ID` и `TELEGRAM_API_HASH`.
3. Запустите:
   ```bash
   python check_telegram_account.py
   ```

Файл `.env` не попадает в репозиторий (указан в `.gitignore`).

### Интерактивный ввод

Если в `.env` (или в переменных окружения) нет `TELEGRAM_API_ID` и `TELEGRAM_API_HASH`, скрипт запросит их в консоли. При первом запуске также понадобятся номер телефона и код из Telegram (и пароль 2FA, если включён). Сессия сохраняется в `check_session.session`; при следующих запусках повторный ввод не нужен.

## Пример вывода при успехе

```
--- Подключение успешно ---
ID:       123456789
Username: @example
Имя:      Иван Иванов
Телефон:  (используется для входа, в API не отображается)
```

## Основные ошибки

| Ситуация | Сообщение |
|----------|-----------|
| Неверные api_id/api_hash | «Неверные api_id или api_hash. Проверьте данные на my.telegram.org» |
| Нет сети | «Ошибка сети» + совет проверить интернет |
| Ограничение Telegram (FloodWait) | «Подождите N сек. и попробуйте снова» |
| Сессия недействительна | «Удалите файл check_session.session и войдите заново» |

## Важно

- Файл `check_session.session` содержит авторизационные данные — не публикуйте его.
- При смене api_id/api_hash или номера телефона удалите `check_session.session` и выполните вход снова.
